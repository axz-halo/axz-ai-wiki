L1 SEMINAR · 30분 실습

오늘 30분, Claude로 와이어프레임까지

이론 챕터를 다 읽었으니 이제 손을 움직입니다. 클로드 세팅 5분 + 200% 원칙 5분 + 직무 커스터마이징 2분 + 와이어프레임 10분 실습 + Figma 내보내기 5분 + 응용 변주 3분. 모두 합쳐 30분 안에 한 사이클을 끝냅니다.

대상 · 진행 방식
PM·디자이너·주니어 개발 등 비개발 인접 직군 위주. 본책 1-1~1-6을 한 번 훑은 분에게 적합합니다. 세미나 현장에선 강사 1명 + 참석자 8~12명 기준으로 진행되며, 자율 학습이면 한 시간 정도로 페이스 조절을 권장합니다.
이 세션의 흐름
  1. PART 1 맛보기 (12분) — 세팅·200% 원칙·직무 커스터마이징
  2. PART 2 실전 응용 (15분) — Skills로 와이어프레임 10분
  3. 응용 변주 3가지 + 다음 회기 예고
  4. 이 세미나에서 소개한 도구

PART 1 · 맛보기 (12분)

1.1 클로드 세팅 5분

이미 본책 1-1에서 한 번 다뤘지만, 실습 직전에 한 번 더 점검합니다. 세팅 차이로 인해 같은 프롬프트가 사람마다 다른 결과를 내는 경우가 많습니다.

3단계 세팅 체크

01 계정·인증 · 사내 SSO로 로그인. 개인 Gmail은 절대 금지. 사내 계정에는 감사로그·DLP가 자동 연동되어 법무·보안 요건을 충족합니다.

02 표시 설정 · 다크/라이트 테마 결정 · 코드 폰트 확인 · 한국어 인터페이스 옵션 점검. 세미나 화면 공유 시 라이트 모드가 일반적으로 가독성이 좋습니다.

03 PDF·검색 기능 활성화 · 기본은 켜져 있으니 그대로 두되, 정보 분류 등급 C 이상 문서를 다룰 때는 검색 기능이 외부 호출하지 않는지 사전 확인.

팁 · 세미나 첫 5분 자가 점검
실습 시작 전 참가자 모두 동일한 모델(Sonnet 4.6) · 같은 인터페이스 언어를 맞추세요. 결과 비교가 가능한 환경이 만들어져야 학습 효과가 큽니다.

1.2 200% 원칙 · 돈 아깝지 않게 쓰는 3가지

좋은 모델을 비싸게 쓰는 것이 아니라, 적절한 모델을 효율적으로 쓰는 것이 200%를 만듭니다. 세 가지 원칙을 외워 두세요.

원칙실행 방법
01 모델은 단계에 따라 바꾼다탐색은 Sonnet, 최종 교정만 Opus, 대량 변환은 Haiku.
02 컨텍스트는 최소·필수만긴 PDF 전문을 통째로 붙이지 말고 필요한 페이지·표만 발췌해 붙임.
03 응답 캐싱·재요청 활용같은 작업을 반복 호출하지 말고 첫 응답을 저장 후 변형 요청으로 비용 절감.
−55%
3원칙 준수 시 토큰 사용량
×2.1
응답 만족도 (같은 시간 대비)
−70%
반복 작업 평균 시간

1.3 첫 프롬프트 한 발 · 직무 커스터마이징

모든 직무가 똑같이 시작하지 않습니다. 5요소 골격 위에 직무별 첫 한 줄을 다르게 잡으면 답변 톤이 즉시 달라집니다.

2-3-2 직무별 첫 줄 (Role Prompting 응용)
─────────────────────────────────────────
PM       → "10년차 PM이고 모호한 아이디어를 PRD로 확장한다."
디자이너 → "AXZ 디자인 시스템에 익숙한 시니어 디자이너다."
개발자   → "시니어 엔지니어. 단계별로 생각해서 가설 3개부터 제시."
마케터   → "B2C 그로스 마케터. 채널별 톤을 분리해 카피를 짠다."
HR       → "10년차 HR 파트너. BEI(STAR) 질문을 잘 짠다."
세미나 팁
참가자 본인 직무의 한 줄을 직접 적어 보세요. 그 한 줄이 이후 모든 응답 톤을 결정합니다. "10년차"는 자신감 톤, "시니어"는 디테일 톤을 만듭니다.

PART 2 · 실전 응용 (15분)

Claude Design + Artifacts + Claude Skills를 통해 한 가지 목표 — 모바일 로그인·회원가입·비밀번호 찾기 와이어프레임 3안 생성 — 을 10분 안에 완주합니다.

2.1 오늘의 목표 · 완성물 스펙

항목스펙
대상 화면로그인 · 회원가입 · 비밀번호 찾기 (모바일 기준)
완성 기준3안 동시 생성 · Figma 내보내기 가능 · 데모용 텍스트 포함
제약실서비스용 디자인 시스템 미적용 · 와이어프레임 수준
제한 시간10분 (조합 · 수정 누적 포함)

2.2 단계 ① 프롬프트 4줄로 3안 생성

[역할] AXZ 디자인 시스템에 익숙한 시니어 모바일 디자이너.
[맥락] B2C 서비스. 로그인 · 회원가입 · 비밀번호 찾기 3개 화면.
[지시] 와이어프레임 3안을 동시에 생성. 각 안은 컴포넌트·간격·톤을 달리.
[형식]
  A안: 정통형 · 폼 중심
  B안: 모던형 · 카드 분리
  C안: 미니멀 · 텍스트 최소
FIGURE L1-P · 와이어프레임 3안 생성 4단계

2.3 단계 ② 결과 검토 · 5분 룰

첫 생성물은 비판하지 마세요. 5분만 보고 다음 결정을 내립니다. 다섯 항목으로 빠르게 점검합니다.

✓ 레이아웃 — 정보 위계·간격·균형이 보이는가

✓ 컴포넌트 — 버튼 · 입력 · 안내 텍스트 자리가 맞는가

✓ 톤 일관성 — 화면 3개의 톤·간격이 시리즈로 묶이는가

✓ 디자인 시스템 후보 — 우리 시스템 토큰으로 옮기기 적합한가

✓ 비고 — 어드민 · 에러 등 결손 케이스가 노출되는가

2.4 단계 ③ 조합 요청 · 수정 누적 프롬프트

첫 생성물에 만족하지 마세요. "더 좋게" 대신 "이 부분을 이렇게" 같이 구체적으로 조합합니다.

[조합] 다음 조건으로 3안 중 A안의 레이아웃 · B안의 컴포넌트 · C안의 톤을
       하나로 합쳐 새 안을 생성.
[수정] 미니멀 톤 유지 · 컴포넌트 사이 간격 32px · 톤 톤매뉴얼 적용
[제약] 한국어 · 모바일 폭 · 다크모드는 다음 회기로 미룸
[형식] 와이어프레임 1안 (최종 통합본)
자주 하는 실수
"더 예쁘게" "더 깔끔하게"는 결과가 들쭉날쭉합니다. AI에겐 의식 흐름을 맡기지 말고, 의식의 단어를 직접 주세요. "32px 간격", "그림자 제거", "버튼 모서리 4px" 같이.

2.5 단계 ④ Figma 내보내기 · 팀 공유까지

AI가 만든 결과는 팀이 볼 수 있어야 의미가 있습니다. 한 클릭으로 팀 Figma 파일로 보내고, 슬랙에 공유합니다.

3단계 팀 공유

① Figma export → 별도 브랜치 파일로 · Claude Design 출력의 "Figma 내보내기" 클릭. 팀 메인 파일이 아닌 브랜치 파일로.

② 코멘트 첫 시드 · 본인이 직접 "A안 시도. 톤 검토 필요" 같은 한 줄 코멘트를 남겨 디스커션 시작점 제공.

③ 슬랙 공유 텍스트 표준 · "와이어프레임 3안 → #design-review · 검토 후 결정 부탁드립니다." 한 줄로.
