모델 선택 결정 트리

세 모델은 같은 API를 쓰지만 속도·가격·추론 깊이가 다릅니다. 업무 성격에 따라 골라 쓰면 시간과 비용이 동시에 줄어듭니다. 이 챕터는 라인업·결정 트리·혼용 전략·실전 케이스 3선을 한꺼번에 다룹니다.

2026-04 라인업

Opus 4.7은 복잡한 추론·긴 문서 분석, Sonnet 4.6은 일상 업무와 코드 리뷰, Haiku 4.5는 빠른 답변과 반복 작업에 적합합니다. 세 모델 모두 200K 토큰의 컨텍스트 창을 공유합니다 (한글 기준 약 120,000자 = 단행본 1권).

Opus 4.7 — 무엇이 달라졌나

2026년 3월 출시. 기존 Opus 4.6 대비 "생각하는 시간이 더 길고, 도구를 더 안정적으로 쓰는" 모델. 특히 다단계 의사결정과 긴 문서 분석에서 체감 차이가 큽니다.

언제 Opus 4.7을 고르는가 — 3가지 장면

① 장문 기획서 분석·요약 · 한글 4,000자 이상 문서에서 일관성·누락 감소가 체감됩니다.

② 여러 결정이 연결된 업무 · "가설 → 평가 → 대안 도출" 같은 다단계 추론에서 4.6 대비 논리 비약이 적습니다.

③ Tool use가 섞인 대화 · 파일·검색·계산 등 여러 도구를 조합하는 분석 작업에서 엉뚱한 도구 호출이 눈에 띄게 줄었습니다.

언제 4.7이 과투자인가
이메일 한 통, 짧은 번역, 회의록 요약 같은 일상 작업에는 Sonnet 4.6이 여전히 가성비가 좋습니다. 전환 체크 한 줄 — "이 답이 틀리면 회의를 다시 해야 하는가?" YES면 4.7.

결정 트리 — 3분 안에 고르기

매번 망설이지 않도록, 다섯 줄 질문으로 정리했습니다. 결정 트리를 거쳐도 결론이 애매하면 Sonnet 4.6이 정답입니다 (사내 사용량의 72%가 Sonnet).

Q1. 응답이 실패하면 업무가 크게 지연되는가? NO → Haiku 4.5 (속도 우선)

Q2. 문서/코드가 2,000단어(한글 4,000자)를 넘는가? NO → Sonnet 4.6

Q3. 다단계 추론(요약 → 평가 → 대안)이 필요한가? YES → Opus 4.7 / NO → Sonnet 4.6

Q4. 같은 작업을 50회 이상 반복하는가? YES → Haiku 4.5 (비용)

Q5. 첫 응답이 완전해야 하는가(재작성 금지)? YES → Opus 4.7

혼용 전략 "Sonnet 먼저, Opus 마지막"

가장 실속 있는 사용 패턴은 두 모델 혼용입니다. 탐색 단계엔 Sonnet으로 여러 번 아이디어를 뽑고, 최종본만 Opus로 한 번 다듬는 방식. 같은 대화 안에서 모델을 바꿔도 맥락은 유지됩니다.

FIGURE 1-3 · 혼용 3단계

Sonnet은 "같이 고민하는 동료", Opus는 "마지막 리뷰하는 선배".

✓ 같은 대화창에서 모델만 바꿉니다 — 맥락 자동 유지.

✓ Opus로 전환할 때 "지금까지의 결정을 요약해 줘"를 먼저 호출.

✓ Opus 응답은 원본과 비교해 diff를 요청 — 누락 방지.

✓ 혼용 1회 대화의 평균 토큰 비용은 단일 Opus 대비 43% 절감.

실전 케이스 3선

CASE 1 · 재무팀

분기 실적 리포트 초안 작성

상황
30쪽 원본 데이터를 임원용 4쪽 요약으로 가공.
전략
Sonnet으로 섹션별 요약 → Opus로 최종 논리 점검. Haiku는 표 재포맷팅에만 사용.
결과
총 소요 시간 6h → 1h 40m. 재작성 0회.
CASE 2 · 마케팅

SNS 카피 20종 A/B 테스트

상황
같은 프로모션 문구를 20가지 톤으로 변주.
전략
Haiku 4.5로 전량 일괄 생성, Sonnet으로 상위 5개 다듬기, Opus는 사용하지 않음.
결과
API 비용 4,200원 → 450원. 문안 품질은 동등.
CASE 3 · 법무

계약서 리뷰 — 독소 조항 탐지

상황
해외 파트너 30쪽 NDA. 한국 법무팀 리뷰 전 1차 필터링.
전략
Opus 4.7 단일 호출. 다단계 추론이 필수.
결과
독소 조항 12건 중 11건 사전 탐지. 법무팀 리뷰 시간 3일 → 1일.
