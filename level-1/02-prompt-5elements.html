프롬프트 5요소

좋은 프롬프트는 긴 설명이 아니라 구조입니다. 다섯 개의 슬롯만 채우면 Claude가 다시 묻지 않고 바로 실행할 수 있습니다. 초급편 전체에서 가장 중요한 챕터입니다.

이 챕터의 목차
  1. 왜 5요소인가
  2. 5요소 공식 표 + 도식
  3. 나쁜 예 vs 좋은 예
  4. 재사용 템플릿 3종
  5. 실전 예시 A · B · C
  6. 실전 오답 노트 5선

왜 5요소인가

프롬프트 엔지니어링이라는 거창한 이름이 붙어 있지만, 실무에서는 다섯 슬롯을 채우는 습관이면 충분합니다. 사내 1,200건의 Claude 대화를 분석한 결과 5요소가 모두 채워진 프롬프트의 첫 응답 만족도는 87%였고, 3개 이하로 채워진 프롬프트는 41%에 그쳤습니다.

87%
5요소 완성 시 만족도
41%
3요소 이하 만족도
2.3회
재작성 평균 횟수 감소

5요소는 Claude에게 주는 업무 의뢰서입니다. 사람에게도 "무엇을, 어떤 톤으로, 어떤 형식으로 달라"라고 말하면 결과가 달라지는 것과 똑같습니다.

5요소 공식 표

FIGURE 1-2 · 5요소 슬롯의 순서와 역할
요소설명좋은 예시생략 시 증상
역할Claude에게 부여할 입장"너는 시니어 에디터야"톤이 중구난방
맥락배경 · 독자 · 목표"독자는 비개발자 임원"전문 용어 난사
지시해야 할 일"아래 보고서를 3문단으로 요약"엉뚱한 작업 수행
제약지켜야 할 규칙"전문 용어 사용 금지"금기어 포함
형식결과물 모양"각 문단 앞에 소제목"문단·불릿 섞임
슬롯 빈 칸 두지 않기
다섯 칸 중 하나라도 비어 있으면 Claude가 추측합니다. 특히 형식은 생략하면 문단·불릿이 제멋대로 섞여 나오니 반드시 명시하세요. 형식 슬롯에 4줄을 써도 길어서 낭비가 아닙니다 — 재작성 1회 비용이 훨씬 비쌉니다.

나쁜 예 vs 좋은 예

BAD

요약 좀 해줘

역할 없음 · 맥락 없음 · 형식 없음. Claude는 원문 길이의 70%쯤으로 재진술하며, 핵심이 빠지고 중복이 섞입니다.

GOOD

5요소 지정 요약

역할(에디터) · 맥락(임원 독자) · 지시(3문단 요약) · 제약(전문용어 금지) · 형식(한 줄 요약 + 결정 사항 + 액션). 첫 응답에서 바로 사용 가능한 요약이 나옵니다.

재사용 템플릿

아래 골격을 사내 Notion Prompt Kit에 복사해 두고 빈칸만 바꿔 쓰세요. 동일한 골격을 반복 사용하면 Claude의 답변 품질도 일관됩니다.

[역할] 너는 ___________ 이다.
[맥락] 이 작업은 __________ 를 위한 것이다.
       대상 독자는 ___________ 이다.
[지시] 아래 내용을 ___________ 해줘.
[제약] ___________ 는 쓰지 마.
       ___________ 는 반드시 포함해.
[형식] ___________ 형태로 출력해.
[입력]
"""
...붙여넣기...
"""

관례 · 대괄호 섹션 라벨([역할], [지시]…)은 Claude가 빠르게 슬롯을 인식하도록 돕는 관례입니다. 영어로 써도 동일하게 작동하지만, 팀 공용 템플릿은 한글 통일이 비개발자 참여율이 높습니다.

실전 예시 A · B · C

예시 A · 회의록 요약

[역할] 너는 임원 미팅 기록을 정리하는 시니어 어시스턴트야.
[맥락] 독자는 해당 미팅에 참석하지 못한 팀장. 다음 주 결정이 필요해.
[지시] 아래 녹취록을 요약해.
[제약] 추측 금지. 불확실한 부분은 "(확인 필요)"로 표기.
[형식]
  ## 결정 사항 (불릿)
  ## 액션 아이템 (담당자 · 마감일)
  ## 후속 질문
[녹취록]
"""
...붙여넣기...
"""

예시 B · 이메일 톤 조정

이미 써 둔 초안을 "부드럽게" 또는 "더 단호하게" 바꿀 때 쓰는 골격입니다. 역할과 형식만 고정하고 톤 키워드만 교체합니다.

[역할] 너는 B2B 고객 커뮤니케이션 에디터야.
[맥락] 납기 지연 사과 메일. 수신자는 파트너사 상급자.
[지시] 아래 초안의 어조를 "정중하지만 단호하게"로 바꿔줘.
[제약] 일정과 수치는 수정 금지. 추가 사과 문장 삽입 금지.
[형식] 수정본 → 수정 이유(불릿 3개) 순으로 답해.

예시 C · 데이터 분류 (Haiku 추천)

엑셀 한 컬럼을 복사해 붙이면 Claude가 카테고리별로 묶어 줍니다. Haiku 4.5를 쓰면 수백 행도 2초 안에 돌아옵니다.

[역할] 너는 CS 티켓 분류기야.
[맥락] B2C 서비스의 지난 주 CS 티켓 제목 목록.
[지시] 각 제목을 다음 카테고리로 분류해.
[제약] 여러 카테고리에 해당하면 주-카테고리 하나만 선택.
       애매하면 "기타"로 분류하고 말미에 사유 한 줄.
[형식] "카테고리 | 제목" 형식의 마크다운 표.
       카테고리: [결제, 로그인, 배송, 환불, 불만, 기타]
팁 · 분류 표 그대로 엑셀에
Claude가 출력한 파이프(|) 표는 엑셀에서 "구분 기호로 분할" 기능으로 한 번에 컬럼 분리됩니다. 시간이 없다면 "쉼표로 구분된 CSV로 줘"로 형식만 바꾸세요.

실전 오답 노트 5선

사내 Q&A에서 가장 자주 반복되는 실수 다섯 개. 각각은 5요소 중 무엇이 빠졌는지로 원인을 설명할 수 있습니다.

증상빠진 슬롯처방
"너무 길게 답변해"형식"3문장 / 200자 이내 / 불릿 5개" 같은 구체적 상한
"전문 용어가 많아"맥락"독자는 비개발자 임원"으로 읽는 사람 명시
"원문에 없는 내용을 지어내"제약"추측 금지 · 확인 필요는 표기" 한 줄 추가
"톤이 너무 딱딱해"역할"친근한 Slack 메시지 톤" 처럼 발화자 정체성 부여
"무엇을 원하는지 다시 물어봐"지시동사 하나로 좁히기 (요약 vs 정리 vs 평가)

이 챕터가 끝나면

✓ 5개 슬롯을 의식하지 않고 자동으로 채울 수 있다.

✓ Notion Prompt Kit에 본인용 템플릿 3개 이상을 저장했다.

✓ 동료의 프롬프트를 보고 어떤 슬롯이 비었는지 30초 안에 지적할 수 있다.

이 세 가지가 되면 다음 챕터 1-3 모델 선택 결정 트리로 넘어가세요.
