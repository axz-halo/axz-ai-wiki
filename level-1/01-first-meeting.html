Claude 첫 만남

Claude는 채팅창만 있으면 바로 쓸 수 있는 업무 동료입니다. 로그인부터 첫 응답까지 5분이면 충분하고, 모델만 올바르게 고르면 첫날부터 이메일·회의 정리·문서 교정에 바로 투입할 수 있습니다.

로그인과 첫 대화

사내 Okta SSO로 claude.ai에 접속한 뒤, 상단 모델 선택기에서 Sonnet 4.6(권장 기본값)을 고르고 대화창에 업무 맥락을 바로 붙여 넣습니다. 첫 대화의 품질이 이후 모든 응답의 기조를 결정하므로, 5분을 투자해 맥락을 제대로 쌓아 두세요.

3단계 첫 진입

01 계정 로그인 · 사내 포털 → Claude 타일 → Okta SSO 승인. 개인 Gmail로 로그인하지 마세요. 사내 계정은 감사로그·DLP·기밀 관리가 연동되어 있습니다.

02 모델 선택 · 처음엔 Sonnet 4.6을 기본으로. 짧은 질문은 Haiku 4.5, 긴 분석은 Opus 4.7로 전환. 같은 대화 도중에도 모델을 바꿀 수 있고, 이전 맥락은 그대로 유지됩니다.

03 첫 프롬프트 · "나는 OO팀 신입이고, 오늘 아래 이메일 초안을 다듬고 싶습니다"처럼 역할과 목적을 한 줄로 알려주세요. 본문을 붙여 넣으면 Claude가 맥락을 자동 분류합니다.

팁 · 대화 제목 바꾸기
대화 제목은 자동 생성됩니다. "[OO팀] 이메일 초안 정리"처럼 프로젝트 태그를 앞에 붙여 직접 수정해 두면 1주일 뒤에도 검색 한 번으로 다시 이어집니다.

claude.ai 화면 구조

UI는 세 영역으로 나뉩니다. 좌측 사이드바, 중앙 대화창, 상단 헤더. 한 번만 좌표계를 잡아 두면 이후 모든 설명이 같은 위치를 가리킵니다.

Projects(프로젝트)가 왜 필요한가

단일 대화는 한 세션 안에서만 맥락이 유지됩니다. 인수인계·분기 OKR 초안·반복되는 QA 양식처럼 여러 주에 걸쳐 같은 지식을 재사용해야 한다면 Projects로 묶으세요. 프로젝트 단위로 지식 파일을 업로드해 두면 그 프로젝트의 모든 대화가 자동으로 해당 자료를 참조합니다.

세 문장 정리
대화는 일회성 주제, 프로젝트는 반복 주제. 지식 파일이 5개 이상 쌓이면 무조건 프로젝트로 승격하세요. 업로드 한도는 프로젝트당 200MB입니다.

첫 5분 체크리스트

✓ Okta SSO로 로그인했고, 개인 계정이 아닌 사내 계정임을 확인했다.

✓ 모델 드롭다운을 열어 세 모델의 이름을 눈으로 확인했다.

✓ 한 번 이상 프롬프트를 보내서 Claude의 응답 속도와 길이를 체감했다.

✓ 대화 제목을 [팀 태그] 형태로 바꿔 저장했다.

✓ 사내 분류 등급 C 이상의 문서는 붙여 넣지 않는다는 원칙을 이해했다.

첫날 자주 겪는 3가지 혼란

혼란처방
"어떤 모델을 고르지?"처음 일주일은 Sonnet 4.6만 쓰세요. 감이 잡힌 뒤 1-3 결정 트리를 따라 바꿉니다.
"너무 길게 답변해서 읽기 힘들어"프롬프트 마지막에 "150자 이내" 같은 길이 제약을 추가.
"대화가 자꾸 맥락을 잊어버려"새 대화를 여는 대신 같은 대화에서 이어가세요. 모델 전환도 맥락 보존.

보안 첫 원칙 — 붙여 넣기 전에 한 번 더

Claude는 사내 규정에 따라 대화 기록을 학습에 쓰지 않도록 계약되어 있지만, 데이터 분류 등급은 여러분이 지켜야 합니다. 아래 세 가지는 무조건 원문이 아닌 요약·익명화 버전으로 붙여 넣으세요.

미공개 재무 수치분기 실적·계약 금액"약 OO억" "두 자릿수 성장률"
