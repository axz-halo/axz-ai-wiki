CHAPTER 1-6 · 30분

Claude Design 첫 걸음

2026년 Q1 출시된 Claude Design은 디자이너·기획자를 위해 화면 시안과 플로우를 만들어 주는 도구입니다. 코드 한 줄 없이 자연어로 목업을 만들 수 있어, 개발 전 커뮤니케이션이 훨씬 빨라집니다.

이 챕터의 목차
  1. 무엇을 할 수 있는가
  2. 3분 첫 목업 만들기
  3. 자연어 프롬프트 패턴 5종
  4. 주의 사항 · 디자이너 검수가 필요한 이유

무엇을 할 수 있는가

4가지 핵심 기능

① 자연어 목업 · "로그인 화면에 구글 OAuth 버튼 추가" 한 줄로 화면 생성.

② 컴포넌트 추천 · 맥락에 맞는 UI 패턴(모달·스텝퍼·탭) 자동 제안.

③ Figma 내보내기 · 원클릭으로 팀 Figma 파일에 컴포넌트·토큰 추가.

④ 플로우 가시화 · 여러 화면을 연결한 사용자 흐름도 자동 생성.

3분 첫 목업 만들기

FIGURE 1-6 · Claude Design 첫 사용 플로우
팁 · 입문자가 쉽게 만드는 법
"~처럼 보이게 해줘"(예: "Toss 송금 화면처럼") 같은 표현이 가장 빠릅니다. 사내 디자인 시스템이 없다면 프롬프트에 "AXZ 기본 톤"을 추가하세요. 첫 목업은 완성도 60%로 받아서, 팀과 얘기하며 나머지 40%를 채우는 것이 시간 절약에 가장 좋습니다.

자연어 프롬프트 패턴 5종

중급편 4장에서 다시 자세히 다루지만, 초급에서도 이 다섯 패턴은 익혀두면 좋습니다.

#패턴예시
01상태 나열형"기본·호버·활성·비활성·에러 5상태를 화면에 함께 표시"
02페르소나 기반"신입 사용자 입장으로 보이게" → 친절·환영 톤 강조
03디자인 시스템 강제"사내 토큰만 사용" → 색·간격 자동 통일
04이전 시안 참조"design-v1 시안과 같은 레이아웃" → 일관성 유지
05수정 요청 누적"방금 만든 화면, 상단 헤더만 좀 더 진하게"

입문자용 한 줄 예시 3가지

온보딩 화면 · "신규 가입 직후 5단계 환영 플로우. 진행 표시 + 건너뛰기."

대시보드 · "KPI 4개·차트 1개·최근 활동 리스트. 임원용 한 화면 분량."

설정 페이지 · "프로필·알림·보안·결제 4 섹션. 좌측 탭, 우측 본문."

주의 사항 · 디자이너 검수가 필요한 이유

아직 자동 배포 금지
Claude Design이 만든 결과를 검토 없이 팀 Figma 메인 파일로 바로 내보내지 마세요. 반드시 별도 브랜치 파일에서 확인 후 머지합니다. 정식 핸드오프 파이프라인은 중급편 4장에서 다룹니다.

BAD

"AI가 만들었으니 그대로 개발에 넘기자"

접근성·실 사용 컨텍스트·디자인 시스템 위반을 디자이너만 잡을 수 있음. 후속 비용이 더 큼.

GOOD

"AI가 60% 만들고, 디자이너가 40% 다듬어 함께 공유"

초안 시간만 압축. 디자이너의 손길은 여전히 필수 검수 단계로 유지.

이 챕터가 끝나면

✓ Claude Design으로 화면 1개를 만들어 Figma로 내보내 봤다.

✓ 다섯 가지 자연어 패턴 중 두 개는 직접 써봤다.

✓ "AI 결과 → 팀 메인 파일 직배포 금지" 원칙을 안다.
