단계 ① 수집 — FeedbackBar

모든 에이전트 응답 하단에 5점 + 자유 기술 입력. 제출은 클릭 1회로 완료합니다. "왜 낮은 점수인지" 선택지를 프리셋으로 제공하면 응답률이 2배 상승합니다.

// FeedbackBar.tsx
export function FeedbackBar({ responseId }: Props) {
  const [rating, setRating] = useState(0);
  const [reason, setReason] = useState("");
  const submit = () =>
    api.feedback.submit({ responseId, rating, reason });
  return (
    <div className="border-t pt-2 flex items-center gap-2">
      <StarPicker value={rating} onChange={setRating} />
      <ReasonChips options={["부정확", "느림", "과장", "유용"]}
                   onSelect={setReason} />
      <button onClick={submit}>제출</button>
    </div>
  );
}

단계 ② 라벨링 — LLM 분류기

수집된 자유 기술을 Claude Haiku 4.5로 자동 라벨링합니다. 5개 라벨만 사용하는 것이 일관성 유지의 핵심입니다.

# feedback_labeler.py
from anthropic import Anthropic

LABELS = ["factual_error", "slow", "overconfident",
          "missed_context", "useful"]

def label(reason: str) -> str:
    res = Anthropic().messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=16,
        system=f"Pick exactly one label from: {LABELS}. "
               "Respond with only the label, nothing else.",
        messages=[{"role": "user", "content": reason}],
    )
    label = res.content[0].text.strip()
    return label if label in LABELS else "missed_context"
라벨 분포 모니터링
factual_error 비율이 주간 10%를 넘으면 해당 태스크 템플릿을 즉시 리뷰합니다. 이 지표 하나가 Golden Set 증식의 최강 선행지표입니다.

단계 ③ 주간 배치 — Golden Set Ingest

매주 월요일 00:00 KST, 지난 주 피드백을 층화 샘플링하여 Golden Set에 편입합니다. 신규 Golden Set은 즉시 회귀 테스트 스위트에 추가됩니다.

# golden_set_ingest.py  (cron: 0 0 * * MON)
from datetime import date, timedelta

def weekly_ingest():
    since = date.today() - timedelta(days=7)
    pool  = fetch_feedback(since)
    sample = stratified_sample(
        pool,
        strata="label",
        per_stratum=8,       # 주 +40 Golden Set
    )
    for item in sample:
        golden_set.add(item)
        regression_suite.add_case(item.prompt, item.expected)
    notify_slack(f"Golden Set +{len(sample)} 건 추가 "
                 f"(누적 {len(golden_set)})")

A/B 기반 회귀 판정

Golden Set 회귀 통과율만으로는 어떤 변경이 퇴행을 유발했는지 알기 어렵습니다. 프롬프트 변경이나 모델 업그레이드 직전엔 반드시 A/B 비교를 돌리세요.

# ab_regression.py
def ab_compare(variant_a: str, variant_b: str, goldens):
    results = {"a": [], "b": []}
    for case in goldens:
        results["a"].append(score(run(variant_a, case), case.expected))
        results["b"].append(score(run(variant_b, case), case.expected))
    return {
        "a_pass": sum(r >= 0.8 for r in results["a"]) / len(goldens),
        "b_pass": sum(r >= 0.8 for r in results["b"]) / len(goldens),
        "delta":  mean(results["b"]) - mean(results["a"]),
    }

릴리즈 직전 체크 없음

"작년에 쓰던 프롬프트인데 고쳤어요" → 모르는 사이 5% 퇴행. 사용자 피드백이 도착할 때까지 2주 블라인드.
