세션 종료 훅

.claude/hooks/on_session_end.sh가 세션 종료 시점에 자동으로 요약·미결 작업·다음 단계 힌트를 HANDOFF.md에 기록합니다.

#!/bin/bash
# on_session_end.sh — Claude Code 세션 종료 훅
claude \
  --prompt "이 세션에서 한 일, 미결 작업, 다음 단계를 150자 내로 요약" \
  --context "$CLAUDE_SESSION_TRANSCRIPT" \
  --output-format markdown \
  > HANDOFF.md

git add HANDOFF.md
git commit -m "session: $(date +%Y-%m-%dT%H:%M) handoff" --no-verify
규칙 · 커밋 컨벤션
HANDOFF.md는 매 PR에 포함되어야 합니다. 리뷰어가 "이 PR이 어디까지 왔는지" 30초 안에 파악할 수 있어야 합니다.

컨텍스트 브리지

다음 세션 시작 시 HANDOFF.md + 에이전트 메모리를 한 번에 주입합니다. LLM에게 "이전 대화 기억하니?"라고 묻지 않고, 시스템 메시지에 통합합니다.

// contextBridge.ts
import { readFile } from "node:fs/promises";

export async function bridgeContext(sessionId: string) {
  const handoff = await readFile("HANDOFF.md", "utf-8");
  const memory  = await readFile(".claude/memory.jsonl", "utf-8");
  const lastPR  = await gh.getLatestPR();
  return {
    systemAugment: [
      "## 이전 세션 HANDOFF", handoff,
      "## 장기 메모리(최근 50건)",
      memory.split("\n").slice(-50).join("\n"),
      "## 최근 PR", lastPR.url + " — " + lastPR.title,
    ].join("\n\n"),
  };
}
토큰 절약
브리지 페이로드는 8K 토큰 이하로 유지하세요. 장기 메모리를 통째로 실으면 실제 작업 공간이 좁아져 답 품질이 떨어집니다.

재개 카드 (Resume Card)

앱을 다시 열었을 때 사용자에게 "어디까지 했는지"를 시각적으로 되짚어 줍니다. Continue · Fresh Start 두 선택지를 명확히 제공합니다.

// SessionResumeCard.tsx
export function SessionResumeCard({ handoff, timeAgo }: Props) {
  return (
    <Card className="border-l-4 border-orange-500 bg-orange-50">
      <CardHeader>
        <span className="text-xs text-orange-600">
          {timeAgo} 전 마지막 작업
        </span>
        <h3 className="font-bold">{handoff.lastTask}</h3>
      </CardHeader>
      <CardBody><p>{handoff.summary}</p></CardBody>
      <CardFooter className="flex gap-2">
        <Button primary>이어서 하기</Button>
        <Button ghost>새로 시작</Button>
      </CardFooter>
    </Card>
  );
}
마이크로 인터랙션
"12분 전" 같은 상대 시간 표기가 절대 시각보다 재개 결정을 빠르게 합니다. 1시간 이상 경과 시 "어제 17:22"로 전환합니다.

장기 메모리 3계층 구조

세션 단위를 넘어 팀·조직 단위로 쌓는 메모리는 보존 기간·갱신 권한이 다른 3계층으로 분리해야 합니다.

개인 메모리

내 프로젝트 · 내 취향. .claude/memory.jsonl. 보존 무기한, 공유 안 함.

팀 메모리

CLAUDE.md · 규칙 · 도메인 용어. Git에 커밋, 팀 PR로 갱신.

조직 메모리

보안 정책 · 브랜드 보이스. 플랫폼팀이 소유. 조직 전역 주입.

속성규칙
우선순위조직 > 팀 > 개인. 충돌 시 상위가 이김.
갱신 주기개인 실시간 · 팀 주 1회 · 조직 분기 1회.
Secret 정책어떤 계층에도 원문 보관 금지. 참조만.

사례 — 신규 입사자 온보딩

CASE · 페이스북-격 사내 사례

온보딩 기간 1주일 → 1.5일

기존엔 신규 입사자가 1주 동안 사내 Dev 샌드박스에서 CLAUDE.md, 코드 가이드, 도메인 용어집을 각자 수동으로 주입. Claude는 사람마다 다른 답을 냈고 "대체 뭘 물어야 하는지"조차 모호.

전환 후: 조직 메모리에 보안 정책·브랜드 보이스 고정, 팀 메모리에 도메인 용어·파이프라인 규칙 고정. 신규 입사자는 개인 메모리만 관리.

HANDOFF.md 안티패턴 5선

안티패턴왜 위험한가
모든 로그 복사채팅 로그를 그대로 붙여넣기. 토큰 폭탄.
