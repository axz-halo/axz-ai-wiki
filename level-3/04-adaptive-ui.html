한 화면이 입문자에겐 친절하고 숙련자에겐 빠릅니다. 숙련도를 3단(beginner · intermediate · expert)으로 추정하고, UI 밀도·툴팁·단축키 노출을 조정합니다.

숙련도 추정 지표

모든 지표는 로컬 저장만 사용합니다. 조직 간 공유되지 않습니다.

레벨세션 수단축키 사용
Beginner5회 미만0~1회
Intermediate5~30회주 3회 이상
Expert30회 초과주 10회 이상
팁 · 휴면 보정
휴면 기간이 14일 이상이면 한 단계 하향합니다. "Expert였는데 한 달 쉬었다"는 상태는 대부분 다시 Intermediate급 힌트가 필요합니다.

숙련도 훅

// useUserSkill.ts
import { useMemo } from "react";

export function useUserSkill() {
  const sessions  = useLocalValue("sessions.count", 0);
  const shortcuts = useLocalValue("shortcuts.weekly", 0);
  const lastSeenDays = useLocalValue("lastSeenDays", 0);

  return useMemo(() => {
    // 14일 휴면 시 한 단계 하향
    const dormancyShift = lastSeenDays > 14 ? 1 : 0;
    if (sessions < 5)  return "beginner";
    if (sessions > 30 && shortcuts >= 10 && !dormancyShift) return "expert";
    return "intermediate";
  }, [sessions, shortcuts, lastSeenDays]);
}

적응형 온보딩 플로우

첫 5회 세션을 Beginner 전용 트랙으로 설계하세요. 첫 경험이 "너무 많은 옵션"이면 이후 평생 Intermediate조차 도달하지 못합니다.

S1 · 가이드 투어 — 3분 인터랙티브 가이드. 좌측 헤더 2개 기능만 노출.

S2-3 · 템플릿 선택 — 6개 카드(이메일·번역·데이터·코드 등) 중 1개 고르면 빈 프롬프트 대신 예시가 채워짐.

S4 · 첫 단축키 노출 — 한 번 성공한 액션에 "⌘↵로 빠르게 실행할 수 있어요" 코치마크.

S5+ · 자동 승급 제안 — "이제 템플릿 없이 바로 써보시겠어요?" 명시적 opt-in.

실제 운영 사례 2종

CASE 1 · 사내 제품 A

Beginner 트랙 도입 효과

3월(도입 전): 첫 세션 완료율 47%. 평균 첫 출력 시간 8분. "어디 클릭해야 할지 모름"이 피드백 상위 1위.

4월(Beginner 트랙 도입 후): 첫 세션 완료율 78%. 평균 첫 출력 시간 2분 40초. "도움말이 자꾸 뜨는데 괜찮음"으로 1위 교체.

CASE 2 · 커뮤니티 제품 B

Expert 옵트인 사용 패턴

사용 21회 넘긴 사용자에게 첫 승급 제안. "⌘K 외 단축키를 모두 숨기고, 파일 쓰기 확인 다이얼로그도 대부분 생략합니다" 문구 명시.

누른 사용자 68%, 그 중 14일 사용 후 회귀 3%. Expert UI가 실제로 원하는 사람에게만 적용된 숫자.
