L2 SEMINAR · 45분 실습

Cowork과 나만의 Skill로 프로토타입 띄우기

중급 본책이 "Claude Code와 MCP 커넥터"를 체계화했다면, 이 세션은 Cowork 환경에서 CMS·B2C 프로토타입을 직접 띄우고 DB 값을 설정하는 과정을 손으로 따라가는 실습입니다. 마지막엔 이 흐름을 나만의 Skill로 묶어 팀 자산으로 만듭니다.

대상 · 진행 방식
기획·PM·디자이너·주니어 개발. 본책 2-1~2-5를 한 번 훑은 분에게 적합합니다. 운영 시간은 Part 1(15분) + Part 2(25분) + Q&A(5분) = 총 45분.
이 세션의 흐름
  1. PART 1 맛보기 (15분) — Cowork과 나만의 Skill
  2. PART 2 실전 응용 (25분) — CMS 프로토타입 + DB 설정
  3. 단계 ⑤ 나만의 Skill로 재사용
  4. 이 세미나에서 소개한 도구

PART 1 · 맛보기 (15분)

1.1 Cowork 30초 소개

Cowork은 비개발자가 자기 폴더·파일을 AI에 맡기고 함께 일하는 데스크톱 환경입니다. 터미널이 없어도, 코드 지식이 얕아도, 파일 생성·편집·배포 준비를 한 자리에서 진행할 수 있습니다.

1.2 Claude Code vs Cowork · 역할 구분

구분Claude CodeCowork
주 대상개발자·DevOps비개발자 + 하이브리드 팀
인터페이스터미널 (CLI)데스크톱 앱
주력 작업코드 리뷰·테스트·배포·리팩터문서·데이터·프로토타입·파일 관리
파일 권한레포 전체 + 훅 제어"선택한 폴더"만 · 샌드박스 기본
플러그인MCP · 슬래시 · 서브에이전트 · 훅MCP · Skills · 플러그인 공통 사용
배포 타겟프로덕션 릴리즈프로토타입 · 내부 공유 중심
둘을 같이 쓰는 것이 정답
기획자는 Cowork으로 프로토타입을 빠르게 띄우고, 개발자는 Claude Code로 같은 레포에서 품질·성능·배포를 책임집니다. 동일 CLAUDE.md를 공유하면 두 환경에서 일관된 톤이 유지됩니다.

1.3 Cowork에서만 되는 3가지

01 선택한 폴더 = 작업 샌드박스 · 사용자가 선택한 폴더 바깥에는 접근하지 않음. "어디까지 만질지" 명시적.

02 Skills · Artifact · MCP 한 자리 · docx·pptx·xlsx·pdf 같은 공식 Skill, 플러그인 Skill, MCP 커넥터가 모두 같은 대화 창에서 호출.

03 파일 "링크"로 성과를 즉시 공유 · computer:// 링크로 완성물을 바로 열어볼 수 있음. 긴 설명보다 빠른 피드백.

1.4 나만의 Skill · 3단계로 만들기

팀에서 반복되는 프롬프트·절차를 SKILL.md 한 파일로 묶으면, Claude가 상황을 인식해 자동으로 해당 절차를 따릅니다. 슬래시 명령보다 가볍고, 프롬프트 복붙보다 영속적입니다.

# 주간 리포트 작성 가이드
## 섹션
1. 이번 주 핵심 지표 (가입·MAU·문의 수)

3. 다음 주 준비
4. 위험·이슈
## 스타일
- 불릿 대신 짧은 문단.
- 숫자는 전주 대비 증감 병기.
- 모든 수치는 [출처: 파일·시트] 주석.
FIGURE L2-P · SKILL 작성 → 저장 → 호출 3단계
description이 Skill의 생사를 가른다
Skill이 자동으로 불리는 이유는 description이 현재 요청과 일치하기 때문입니다. "언제(When), 무엇(What), 누구를 위해(Who)"가 한 줄에 들어가 있어야 Claude가 정확히 선택합니다.

PART 2 · 실전 응용 (25분)

Cowork으로 커뮤니티 팀이 실제 쓸 법한 "이벤트 공지 CMS"를 프로토타입으로 띄우고, 샘플 DB 값을 직접 설정해 팀에 링크 공유하는 지점까지 25분 안에 완주합니다.

2.1 목표물 스펙 · 한 장 요약

항목스펙
제품명AXZ 커뮤니티 이벤트 CMS (프로토타입)
B2C 측이벤트 목록 · 상세 · 신청 폼
어드민 측이벤트 생성·수정 · 신청자 엑셀 다운로드
데이터이벤트 10건 · 신청자 50건 시드 (Cowork이 자동 생성)
스택Next.js + SQLite (팀 데모용 · 단일 파일 DB)
배포로컬 포트 미리보기 → Vercel Preview 링크 공유 (옵션)
제한 시간25분 · 세미나 진행자 화면 공유 기준
프로토타입 범위
오늘의 목표는 "팀이 감을 잡을 만한 수준"이지 배포용 완성품이 아닙니다. 인증·결제·법적 문구는 생략. 실 서비스 전환은 개발 팀의 정식 레포 이관 절차를 따릅니다.

2.2 단계 ① 폴더 연결 + 요구사항 공유

Cowork에 빈 폴더를 연결하고, 다음 요구사항 한 블록을 던집니다. 기술 스택·파일 구조·완성 기준 3가지를 명시하는 것이 핵심입니다.

프로젝트: AXZ 커뮤니티 이벤트 CMS 프로토타입.
기술 스택: Next.js 14 (App Router) + SQLite + Prisma + Tailwind.

폴더 구조 만들기:
  /app         — 공개 페이지 · 라우트
  /app/admin   — 어드민 페이지
  /prisma      — 스키마 · 마이그레이션
  /lib/db.ts   — 접근 유틸

필수 페이지:
  GET  /                  이벤트 목록
  GET  /event/[id]        이벤트 상세 + 신청 폼
  GET  /admin             이벤트 리스트 CRUD
  GET  /admin/applicants  신청자 리스트 + CSV 다운로드

완성 기준:
  1) npm run dev 로 3000 포트에서 렌더링
  2) 시드 데이터가 들어간 상태로 뜨기
  3) 어드민에서 이벤트 생성 → 공개 목록에 즉시 반영
왜 이 구조인가
"요구사항 한 덩어리"로 던져도 Claude는 알아서 파일을 만들지만, 완성 기준 3줄을 적어주면 "완성했다"고 말할 시점의 기준이 명확해집니다. 이 차이가 10분과 30분을 가릅니다.

2.3 단계 ② 스캐폴딩 관찰 · 중간 개입

Cowork은 파일 생성·명령 실행을 한 번에 수십 건 처리합니다. 중간에 개입할 포인트는 3곳입니다.

① package.json 생성 직후 · 의존성 목록이 합리적인지 빠르게 훑는다. 쓸데없는 라이브러리를 쳐내면 충돌이 없다.

② Prisma 스키마 작성 직후 · Event·Applicant 테이블 필드 확인. 마이그레이션 이후엔 바꾸는 비용이 커진다.

③ npm run dev 직전 · "시드 데이터 10건 · 신청자 50건을 자동 생성하는 스크립트도 함께 실행해" 한 줄 추가.

2.4 단계 ③ DB 값 설정 · 자연어로 SQL 쓰기

프로토타입이 떠 있는 상태에서 "데이터를 자연어로 바꿔달라"고 요청합니다. Cowork이 Prisma 쿼리나 SQL을 생성해 실행해 줍니다.

자연어 지시Cowork이 생성하는 쿼리
"다음 주 이벤트 2건 만들고 각각 신청자 5명씩 채워줘"INSERT Event × 2 · INSERT Applicant × 10
"제목에 '취소' 들어간 이벤트는 status를 cancelled로"UPDATE Event WHERE title LIKE '%취소%'
"신청자 중 이메일이 중복된 건 하나만 남기고 삭제"DELETE Applicant GROUP BY email HAVING COUNT > 1
실제 운영 DB에 절대 연결 금지
오늘 실습은 로컬 SQLite 파일 단일 DB입니다. 프로덕션·스테이징 DB 비밀번호·키는 Cowork 대화창에 절대 붙여 넣지 않습니다. 운영 데이터 이관은 DBA의 공식 절차를 거쳐야 합니다.

2.5 단계 ④ 프리뷰 배포 · 팀 공유

로컬에서만 뜨는 결과는 팀이 볼 수 없으니, Vercel Preview 링크로 공유하는 것이 가장 빠릅니다. 이 과정도 Cowork이 대신 수행합니다.

vercel preview 배포 준비:
  - .env.example · README · package.json 정리
  - SQLite 대신 libsql 원격 URL로 스위치 (환경변수)
  - vercel deploy --prebuilt

Preview URL 받으면 그대로 알려줘.

팀에 공유하는 슬랙 메시지 템플릿

왜 Preview 링크인가
같은 링크로 PM·디자인·개발이 동시에 보면서 코멘트 달 수 있어 "설명의 설명" 없이 다음 스프린트 재료가 됩니다. 실 배포는 이 링크를 보고 팀 논의가 끝난 뒤에 개발 팀이 정식 레포로 이관합니다.

## 단계
1. 요구사항 파싱 → Prisma 스키마 초안
2. 스캐폴딩 (App Router · Tailwind · Prisma)
3. 시드 데이터 자동 생성 (한국어 가상 데이터 10·50건)

## 고정 규칙

- 운영 DB 주소·비밀번호는 절대 수용 금지
- 생성 후 CLAUDE.md 한 줄 요약 남기기
Skill이 팀 자산이 되는 순간
이 SKILL.md를 사내 axz-plugins 레포에 커밋하면 다음 달엔 다른 팀도 "프로토타입 만들어줘" 한 줄로 같은 품질의 결과를 얻습니다. 개인 기술이 조직 근육이 되는 전환점입니다.
