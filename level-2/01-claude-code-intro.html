Claude Code 입문

Claude Code는 Claude와 동일한 모델을 쓰는 CLI입니다. 사내 보안 정책 위에서 파일을 직접 수정하고 Git에 커밋하고 사내 MCP 커넥터로 연결할 수 있습니다. 이 챕터는 설치부터 첫 커밋까지의 전 과정을 다룹니다.

이 챕터의 목차
  1. 왜 Claude Code인가
  2. CLI 설치와 인증
  3. 프로젝트 초기화 — /init과 CLAUDE.md
  4. 첫 Git 커밋까지
  5. tool use 승인 흐름
  6. 첫날 권장 워크플로우

왜 Claude Code인가

본격적인 자동화로 가기 위한 도구입니다. Claude Code는 모델뿐 아니라 파일·Git·터미널을 직접 조작합니다. 이 차이가 반복 업무의 결정적 단축을 만듭니다.

claude.ai 브라우저

채팅창에 컨텍스트를 붙이고 즉답을 받는 단계. 재사용은 복붙에 의존.

Claude Code 터미널

Claude가 파일을 직접 읽고 쓰고 명령을 실행합니다. 슬래시 커맨드로 반복 업무를 한 단어로 줄임.

Claude Code는 Claude에게 키보드를 맡기는 것입니다. 여러분은 감독·리뷰·결정만 하세요.

CLI 설치와 인증

설치 환경은 macOS, WSL2, 또는 Linux 컨테이너를 권장합니다. macOS 기준으로 WSL2/Linux 환경도 동일한 절차를 따릅니다.

01 Node.js 20 LTS 설치 확인

설치 표준은 Node 20 LTS입니다. node -v로 v20.x를 확인하고, 다르면 nvm install 20으로 갱신.

02 Claude Code 전역 설치

npm i -g @anthropic-ai/claude-code
claude --version

03 사내 SSO 인증

터미널에서 claude를 처음 실행하면 브라우저가 자동으로 열립니다. Okta SSO로 인증하면 OAuth 토큰이 ~/.claude/credentials.json에 저장됩니다.

$ claude
# 브라우저 자동 실행 → Okta SSO → 토큰 저장
# 이후 명령은 자동 인증
claude --authenticated
주의 · 토큰 보관
credentials.json은 개인 토큰입니다. Git에 커밋하거나 Slack에 첨부하면 즉시 만료 + 보고 사항입니다. 의심 시 claude logout으로 즉시 폐기.

프로젝트 초기화 — /init과 CLAUDE.md

Claude Code는 CLAUDE.md를 프로젝트 컨텍스트 파일로 씁니다. 기존 프로젝트에서 /init 슬래시 커맨드를 실행하면 자동 작성을 시작합니다.

$ cd ~/work/my-project
$ claude
> /init
# → CLAUDE.md 자동 작성 시작
# → 기본 스택, 빌드 명령, 커밋 컨벤션, 사내 디렉터리 구조 파악

작성된 CLAUDE.md는 사내 시크릿 가드라인에 따라 수정해 완성하세요. 다음 모든 세션에서 Claude Code는 이 파일을 먼저 읽어 일관된 톤을 유지합니다.

팁 · CLAUDE.md는 살아있는 문서
새 라이브러리, 새 CI 컨벤션, 디렉터리 규칙이 바뀔 때마다 갱신하세요. 사내 컨벤션, DB 마이그레이션 시 이슈도 함께 작성해 두면 좋습니다.

초기화 후 10분 체크

✓ CLAUDE.md 자동 작성 완료를 확인했다.

✓ .gitignore, .editorconfig, README 등 핵심이 다 잡혔다.

✓ .claude/settings.local.json이 .gitignore에 등록됐다.

✓ claude mcp list로 기존 커넥터 운영 확인.

✓ 사내 디렉터리 구조가 정확히 인식됐다.

첫 Git 커밋까지

설치 직후 30분 안에 README에 한 줄 추가해 커밋해 보세요. Claude Code의 파일 수정 → 커밋 흐름을 한 번 경험하면 이후 작업 부담이 확 줄어듭니다.

> README.md 끝에 "## Setup · Updated by AXZ AI 2026-04-17" 라인 추가하고
  "docs: refresh setup heading" 커밋해줘
[Claude 제안 표시]
  - Edit Tool 호출 → 파일 수정 예시
  - Bash Tool 호출 → git add, git commit 명령
  - 승인: 각 단계마다 y2/M/I

첫날 권장 워크플로우

모든 명령을 완벽히 이해하지 않아도 괜찮습니다. 2시간 슬래시 커맨드로 감을 잡는 것이 목표입니다.

시간해야 할 것
00:00 · 설치Node 20 + Claude Code 설치 + SSO 인증 + claude --version
00:15 · 초기화기존 프로젝트에서 /init 실행 → CLAUDE.md 점검
00:30 · 체험README 추가 + 첫 커밋 + tool use 승인 흐름 체감
01:00 · 탐색/review 실행 → 가장 작은 PR로 결과 확인
01:30 · 연결Slack · Notion 커넥터 2개만 먼저 연결
02:00 · 기록오늘 배운 점을 ~/.claude/CLAUDE.md에 저장

이 챕터가 끝나면

✓ claude --version이 정상 응답한다.

✓ 사내 SSO 인증을 한 번 통과했다.

✓ /init으로 CLAUDE.md가 생성됐다.

✓ README 한 줄 수정 + 커밋을 Claude Code로 해봤다.

✓ tool use 승인 흐름 3단계를 머리에 새겼다.
