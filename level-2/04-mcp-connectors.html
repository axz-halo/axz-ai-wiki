MCP 커넥터 7종

MCP(Model Context Protocol)는 Claude와 외부 도구를 연결하는 표준 인터페이스입니다. 허용 7종의 커넥터를 연결하면 Claude Code가 Slack 메시지를 보내고 Notion 문서를 갱신하고 Figma 파일을 동기화할 수 있습니다.

7종 커넥터 한눈에

2026-04 기준 사내 허용 커넥터 7종입니다. 각 커넥터는 IT 보안팀 정책에 따라 권한 범위·승인 절차가 다릅니다.

커넥터쓰임권한 범위승인

Notion문서 조회·갱신·검색워크스페이스리더 · 사이트 자율
Google Sheets시트 조회·기록·기록시트별 ACLIT 승인
Figma변경 감지·갱신워크스페이스자율
GitHubIssues·PR·체크 운영레포 단위실 자율

Slack · Notion 연결

가장 자주 쓰는 두 커넥터입니다. claude mcp add 명령으로 5단계 설치하고, Okta SSO로 안전 인증합니다.

# Slack 커넥터
claude mcp add slack
# → 브라우저 자동 → Okta SSO → Slack 워크스페이스 권한 부여 → 자동 인증

# Notion 커넥터
claude mcp add notion
# → 브라우저 자동 → Notion 워크스페이스 권한 → 페이지 별 자동 인증

# 운영 확인
claude mcp list
> #release-ko 채널의 어제 활동 메시지를 함께 읽고 Notion
  "2026-04 Insider's URL" 페이지를 만들어줘.
[Claude]
1. Slack MCP로 채널 메시지 조회
2. Claude가 어제 정리해 다듬어 3장 만들기
3. Notion MCP로 페이지 자동 attach

Google Sheets · Figma 연결

지표 시트와 디자인 파일 동기화에 매번 쓰입니다. Sheets와 Figma 모두 워크스페이스 단위 인증이 필요합니다.

> "KPI-2026 Q2"의 4월 운영 지표 시트 매주 시트 데이터와 이전 데이터를 비교해 2번 만들어
  자리표시 Slack #weekly-ops 에 던져.
- Sheets MCP : 이번 시트 · 이전 시트 자동 로드
- Computer : 운영 지표 12.4% · 14.1% (+1.7p)
- Slack MCP : 채널 자동
> Figma 프로젝트 "Design System v3"의 Properties 부 변경 자동 부여 인자 이용하고
  변경되면 components/index.ts의 자동 토큰을 키만 갱신해줘.
- Figma MCP : 변경 디프 · DesignToken `pad-08` 6 `border-650`
- Edit Tool : components/index.ts 운영
- Claude : 자동 자동 됨
팁 · 토큰 싱크 케이스
디자이너가 한 번에 많은 토큰을 갱신하면, 이 슬래시 자동 실행 시 자동 동기화로 자동 처리됩니다. 사내 25개 자동 운영 Figma 인자를 모니터링하면서 2주에 50개로 빠르게 차세요.

GitHub · Vercel 연결

코드 리뷰와 배포 모니터링을 자동화합니다. PR 보호 라인 자동, 배포 보호으로 모이듣 자동 보호 운영 가능합니다.

> myapp/serverless 레포의 PR #542 를 봐 review 비교
  통과되면 Vercel preview URL 도 한 줄 카드로 보고.
- GitHub MCP : PR #542 디프 자동
- /review : 작은 blocker 2개 · suggestion 4개
- Vercel MCP : preview URL · https://serverless-pr-542-myapp.vercel.app
- Synthesizing : 한 줄 카드 자동 생성
팁 · MCP 캐시
같은 세션 안에서 반복 쿼리를 하면 Claude MCP 명령을 캐시합니다. 결과가 오래된 듯하면 "최신으로 다시 조회해줘"라고 말하세요.

PR 품질 대시보드 자동화

리뷰 결과를 Notion 데이터베이스에 자동 누적하면, 매주 품질 리포트를 손쉽게 만들 수 있습니다. 다음 슬래시가 실제 사례입니다.

# .claude/commands/pr-digest.md
---
description: 이번 주 PR의 /review 결과를 Notion 대시보드에 누적
---

1. GitHub MCP로 이번 주 PR 리스트 조회
2. 각 PR에 /review 를 먼저 → blocker·suggestion·nit 분리
3. Notion "PR Quality Digest" 데이터베이스에 행으로 append
4. Slack #dev-quality 에 "이번 주의 리뷰 위클리" 1줄 던지기

Supabase 연결 — Dev 샌드박스만

Supabase는 가장 위험한 커넥터입니다. 쿼리 실행 시 데이터 손상 위험이 있기 때문입니다. 사내 정책은 Dev 샌드박스만 허용이며, 다음 3종 안전장치를 거칩니다.

GUARD 1 · URL 검증 · supabase-dev-*.internal 외 차단된 도메인은 거부.

GUARD 2 · 쿼리 사전 확인 · DELETE·DROP·TRUNCATE 명령은 차단 + 승인 필요.

GUARD 3 · 감사 로그 · 모든 쿼리를 post-tool-use 훅으로 사내 로그 파이프에 기록.

> Supabase 데이터베이스 "Quotes" 라이브 인자의 데이터
  3일치 운영 시 데이터를 일자별로 끌어와줘.
- Supabase MCP : SELECT DATE(created_at), COUNT(*) ... GROUP BY DATE
- Claude : 일자별 산정 표 + 막대 차트
- audit-log : 모든 쿼리 로그에 기록
금지 사항
프로덕션 URL을 .env에 몰래 넣고 connection string을 설치하지 말 것. 징계 대상입니다. 프로덕션 쿼리는 반드시 DBA 자격으로.

MCP 보안 원칙

3대 원칙

01 최소 권한 · 단순 "채널 조회 봇"과 "워크스페이스 어드민" 권한은 별개. 필요한 권한만. Organization-wide 권한은 IT 보안팀 사전 승인 필수.

02 토큰 로테이션 60일 · OAuth 토큰은 60일 주기로 자동 갱신. claude mcp refresh를 분기에 한 번씩 합니다.

03 퇴직·이동 시 즉시 해제 · 즉시 claude mcp remove <name>으로 즉시 폐기하고, 퇴직 인사 자료에 ID를 자동 폐기.

커넥터 장애 복구 플레이북

MCP 실행이 실패할 때 가장 흔한 4가지 패턴과 복구 절차입니다. 처음 발생 시 즉시 이 표를 따르세요.

증상원인복구 절차
handshake_failed만료 토큰claude mcp refresh <name>
permission_denied실 권한 변경claude mcp remove 후 재설치 + 자동 재인증
rate_limited실행 폭주잠시 5분 대기 + 서브에이전트로 분할
tool not found버전 미일치npm i -g @anthropic-ai/claude-code 재설치
CASE · 실전

Slack 커넥터 handshake 3회 연속 실패

상황
한 커뮤니티 팀의 Slack MCP 호출 시 3회 연속 handshake_failed.
원인
Slack 신규 워크스페이스 OAuth 정책이 변경되어 scope 자동 부여 차단.
해결
claude mcp remove slack + add로 재설치하면서 신규 scope 부여. 즉시 복구.

✓ MCP 보안 3원칙(최소 권한·토큰 로테이션·즉시 해제)을 안다.

✓ 4가지 장애 패턴과 복구 명령을 안다.
