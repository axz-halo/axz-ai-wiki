세션 관리 · CLAUDE.md

Claude Code의 능력은 결국 어떤 맥락을 얼마나 오래 유지하는가에 결정됩니다. 세션·CLAUDE.md·메모리 규칙을 정확히 이해하면 코드 리뷰이건 리팩터이건 일관성이 비약적으로 올라갑니다.

세션의 수명 주기

세션은 claude를 실행한 시점부터의 컨텍스트 단위입니다. 한 세션은 무한정 유지되지만, 길어지면 CLAUDE.md를 명시적으로 재진입하는 것이 좋습니다.

언제 4.7로 에스컬레이션하는가

A. 장문·다단계 추론 · 500줄 이상 PR 리뷰, 3개 이상 파일 동시 분석, 마이그레이션 다중 결정.

B. tool_use 조합 · MCP 커넥터 2개 이상을 동시 호출하는 에이전트. 스키마 정확도의 복리 효과.

C. 원인 분석 세션 · 로그·stack trace를 받아 근본 원인을 추정. 4.6보다 환각이 31% 감소합니다.

해석 주의
4.7은 4.6과 동일 가격이지만 Sonnet 대비 약 5배. 일상·rename 같이 단순한 작업엔 Haiku 4.5를 쓰고, 4.7은 세션당 6회 이내로 제한해 비용 통제하세요.

메모리 규칙 3계층

Claude Code의 맥락 저장소는 세 가지 파일이 함께 동작합니다. 각 파일은 역할이 다르며 어떤 정보를 어디에 두느냐가 핵심입니다.

01 프로젝트 — CLAUDE.md

리포지터리에 커밋. 실 컨벤션·코드 구조·금지. 공개 가능한 내용만.

02 개인 — ~/.claude/CLAUDE.md

로컬 홈 디렉토리. 모든 세션에 자동 로드. 개인 작업 컨벤션, 기억할 결정, 자기 코드·이름 같은 것.

03 세션 — /memory 명령

세션 중 "이 사실은 이 작업 동안만 기억해줘"라는 단기 메모리에 저장. /clear와 함께 사라지는 일회용 메모리.

비밀을 메모리에 넣지 마세요
CLAUDE.md는 Git에 커밋되고, 개인 CLAUDE.md도 같이 다 노출됩니다. API 키, DB 비밀번호, 고객 개인정보·민감 데이터에도 절대 메모리에 들어가면 안 됩니다. 시크릿은 환경 변수·시스템 자원 변수만 활용하세요.

팀 공유 CLAUDE.md 전략

프로젝트 CLAUDE.md는 사내 규칙의 "복사 곳"입니다. 새 컨벤션은 도메인 정의 3개 + 3주 동안 추가·갱신되고, 실 컨벤션은 PR 리뷰에서 매주 보일 됩니다.

원칙 1 · 코드 리뷰 대신 규칙 추가. 한 번 지적한 이슈는 CLAUDE.md에 컨벤션으로 추가.

원칙 2 · 규칙은 금지어가 아닌 대안. "X 쓰지 마"보다 "X 대신 Y를 쓰라"가 효과적.

원칙 3 · Current Status는 매주 갱신. 스프린트 시작 시 PM이 라이프스타일 자동.

원칙 4 · 300줄 넘기지 않기. 길어진 Claude의 핵심 단어량이 빠집니다. 짧고 잘 정리된 라인이 핵심.

안티 패턴

CLAUDE.md 안에 모든 API 문서 통째로 복사 · 토큰 비용 비례 + 컨텍스트 디저진.

"가독성 좋게 해"라는 모호한 컨벤션 · 구체적 컨벤션으로.

라이브러리 버전·릴리즈 · 버전 인자 없는 컨벤션은 빠른 시점.

개인 토큰을 프로젝트 파일에 · 자기 CLAUDE.md에 두세요.
