보안 실수 5선과 훅 7종

아래 다섯 사례는 모두 2024–2026년 사내에서 실제 보고된 Claude Code·MCP 보안 사건을 익명화·단순화한 것입니다. 앞 절까지 배운 훅·슬래시로 모두 사전 방지 가능합니다.

이 챕터의 목차
  1. 사례 1 · .env 파일을 그대로 커밋
  2. 사례 2 · CLAUDE.md에 API 토큰 하드코딩
  3. 사례 3 · MCP 커넥터 과도한 권한
  4. 사례 4 · pre-commit 훅 우회
  5. 사례 5 · Slack 커넥터로 고객 메시지 유출
  6. 보안 훅 7종 통합 체크리스트

사례 1 · .env 파일을 그대로 커밋

AS-IS · 무엇이 일어났나

한 PM이 "환경 변수 좀 봐줘" 요청 → 파일에 .env 자동 등록 → Claude의 PR에 .env 들어가 신규 PR 머지 → 3일 후 GitHub 공개 토큰 노출 → AWS 키 폐기 5건.

TO-BE · 예방

pre-tool-use 훅으로 .env·credentials.json·*.pem 패턴을 차단. .gitignore에 등록되지 않으면 즉시 훅이 차단 + 사후 알림.

사례 2 · CLAUDE.md에 API 토큰 하드코딩

AS-IS · 무엇이 일어났나

편의상 CLAUDE.md에 SLACK_TOKEN=xoxb-...로 적음. Git에 커밋 + 다른 리포지터리 참조로 인해 6주 후 발견. 즉시 폐기.

TO-BE · 예방

CLAUDE.md엔 환경 변수 이름만 적음 (예: ${SLACK_TOKEN}). 실제 값은 ~/.config/claude/env 시크릿 매니저로.

사례 3 · MCP 커넥터 과도한 권한

AS-IS · 무엇이 일어났나

Notion 커넥터를 "워크스페이스 어드민" 권한으로 연결 후 Claude 자체 어시스턴트가 다른 페이지를 손대다 잘못 가서 결과 12개를 덮었음. 손해 12건.

TO-BE · 예방

최소 권한 원칙 · 실 프로젝트 DB만 등록·검색 등 운영. 그 이상은 Read-only 자격으로만 등록. 커넥터 권한 검토는 Ch2-4.7 플레이북으로 분기 실시.

사례 4 · pre-commit 훅 우회 --no-verify

AS-IS · 무엇이 일어났나

pre-commit에 "검토 실패" 실행 → 피해 → git commit --no-verify → 결국 PII 자동 커밋 통과/안 비밀 코드 → 4시간 사용. 보고 사항.

TO-BE · 예방

서버사이드 pre-receive 훅을 함께 두기. 클라이언트 우회해도 서버 차단. Claude Code엔 --no-verify 사용 시 "본인 명시적으로 사용하므로 인지하라"를 CLAUDE.md에 둠.

사례 5 · Slack 커넥터로 고객 메시지 유출

AS-IS · 무엇이 일어났나

"릴리즈 #release-prod 채널에 공유해줘" 요청 → Claude 공유 공급 #all-customers 채널에 잘못 가서 → 신규 고객 자료 사용 후 30분에 인지함.

TO-BE · 예방

Slack 커넥터 쓰기 권한은 DM 한정으로 록. 채널 게시는 /publish approve [link] 실행 명시적 승인 슬래시로만. 또 PII 패턴(이메일·전화·주문번호)에 사후 마스킹.
